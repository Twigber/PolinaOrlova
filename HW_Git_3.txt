GitHub. HW_2
1. На локальном репозитории сделать ветки для:
- Postman    - git branch Postman
- Jmeter     - git branch Jmeter
- CheckLists   - git branch CheckLists
- Bag Reports   - git branch BagReports
- SQL     - git branch SQL
- Charles  - git branch Charles
- Mobile testing    - git branch MobileTesting

2. Запушить все ветки на внешний репозиторий    - git push origin --all
3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта

Bug id:
Title:
Environment:
Precondition:
STR:
AR:
ER:
Attachments:


4. Запушить структуру багрепорта на внешний репозиторий

git add .
git commit -m "add bagreport1.txt"
git push -u origin Bagreports

5. Вмержить ветку Bag Reports в Main

Из Main:
git merge Bagreports

6. Запушить main на внешний репозиторий. - git push
7. В ветке CheckLists набросать структуру чек листа.

Build (Версия сборки)
Environment (окружение)
Test date
Tester
Test type
Checking (список проверок)
Result
Ссылка на багрепорт


8. Запушить структуру на внешний репозиторий 

git add .
git commit -m "add checklist.txt"
git push -u origin CheckLists

9. На внешнем репозитории сделать Pull Request ветки CheckLists в main - клик клик)))))
10. Синхронизировать Внешнюю и Локальную ветки Main - git pull
